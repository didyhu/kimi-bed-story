# kimi-bed-story

**睡前故事脚本项目** — 纯文本故事仓库。无应用代码、无图片、无音频。

## 仓库结构

```
kimi-bed-story/
├── stories/              # 故事内容（每个故事一个目录）
│   ├── story_01_塞伯坦的秘密/
│   │   └── story.md
│   ├── story_02_巧克力工厂/
│   │   └── story.md
│   ├── ...
│   ├── 写作规范与一致性手册.md   # 📖 必读：世界观、角色、道具、规则
│   └── 故事历史记录.md          # 📋 出场记录、伏笔、口癖统计
└── AGENTS.md
```

故事目录命名约定：`stories/story_XX_中文名/story.md`（XX 为两位序号，如 01、02）。

## 故事质量标准

### 对标作品：「不一样的卡梅拉」「哆啦A梦」

这些故事有灵魂，不廉价———它们大胆、有趣、有巧思。写作时始终追问：

- **这一集有什么情节是非读不可的？**（不是"公主打败了机器"，而是一个具体、荒诞、不可替代的场景）
- **有没有让孩子笑出声的段落？**（真正的幽默，不是硬挠痒痒）
- **有没有让孩子紧张起来的段落？**（高效的节奏、真正的悬念，不是拖沓的过渡叙述）
- **配角有没有属于自己的高光时刻？**（不是工具人，是有情感弧线的角色）
- **口袋纪念品的诞生方式是否精巧？**（用一句话交代的还是用心设计的？）
- **结尾是否让人期待下一集？**（树屋句不是交作业，是点燃想象力）

完整的情节标准、禁止心态、量化指标见 `stories/写作规范与一致性手册.md` 第三节。

## 写故事流程

### 一、前置准备

通读以下文件（按顺序）：

1. `stories/写作规范与一致性手册.md` — 硬设定、角色状态、道具清单、禁止口癖
2. `stories/故事历史记录.md` — **未使用伏笔清单**（优先回收）
3. 最近 2 集故事全文 — 了解当前语气、节奏和角色状态

### 二、写前自检（5分钟）

动笔前逐项勾选：

```
□ 本集核心冲突是什么？（不是"关机器"，是"为什么必须关"）
□ 机器关闭方案：至少两件已有物品 + 一个障碍或牺牲？
□ "不是X是Y"口癖已经在脑中打过预防针？
□ 纪念品是什么？它怎样与口袋里至少两件已有物品互动？
□ 这一集"只有这里才能出现的荒诞画面"是什么？
□ 有没有俏皮、有趣的对话段落？（不限角色）
□ 配角有没有属于自己的高光时刻？
□ 有没有精巧的情节设计？（伏笔回收/身份反转/信息差喜剧）
```

### 三、写作中

- 严格遵循手册的写作风格规则和禁止口癖
- 参考"不一样的卡梅拉"和"哆啦A梦"的叙事气质：大胆、天马行空、真诚不粉饰
- 对话先行，叙述穿插——孩子用直率对话推进故事
- 纪念品放入口袋时必须与至少两件已有物品产生互动描写

### 四、写后审核

调用**独立子代理**（`@deepseek-pro`）进行质量审核：

1. **提供材料**：故事全文 + `写作规范与一致性手册.md` + 最近 2 集故事全文（用于连续性比对）
2. **子代理按手册退化迹象清单逐项检查**，对每项打勾并评分（1-5 分）
3. **通过标准**：评分 ≥ 4/5 且退化迹象 ≤ 1 条
4. **每次迭代修改后，重新对照退化迹象清单自查**，确认无新增退化后提交子代理
5. **迭代上限 3 轮**：超限时报告未解决项由人工裁决
6. 子代理最终输出：评分、迹象清单、修改总结

### 五、最终步骤（强制）

故事定稿后，**必须**更新以下文件：

1. `stories/写作规范与一致性手册.md` 第四节 — **口袋物品状态**（新增/消耗/状态变化）
2. `stories/故事历史记录.md` — **角色出场记录表**（新增本集一行）
3. `stories/故事历史记录.md` — **已确认事实清单**（新决裂、新伏笔、新永久变更）
4. `stories/故事历史记录.md` — **标记已使用伏笔**，从未使用清单移除

## 质量红线

- 机器关闭必须有"两件核心物品 + 障碍/牺牲"，禁止单纯按按钮
- "不是X是Y"口癖 ≤ 4 次（目标 1-2 次）
- 纪念品放入口袋时与 ≥ 2 件已有物品互动
- 故事字数 ≥ 2000 字
