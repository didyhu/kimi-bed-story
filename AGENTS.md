# AGENTS.md — 睡前故事项目

## 项目概要
- **定位**: 为小朋友创作的睡前故事系列，用熟悉的动漫角色和玩具做跨作品大混搭，对标"不一样的卡梅拉"风格
- **主角**: 善良公主（粉色猪玩偶）+ 海蓝公主（蓝色兔玩偶），双主角
- **核心公式**: 每集独立故事，通常由打屁巫师发明一台机器引起两个世界的混合/冲突，主角们用不同世界的"核心物品"关闭机器
- **输出物**: 每集一个文件夹（`stories/story_NN_标题/`），包含 `story.md` + `cover.png` + `voice.mp3`
- **前端**: `app/` 下是 React+TypeScript+Vite 的展示应用，`npm run dev` 启动，`npm run build` 构建

## 设置文件指引
- 写作规则、角色池、道具状态、退化清单、角色视觉设定、配图 prompt → `stories/写作规范与一致性手册.md`（唯一权威文件）

## 极易犯错的陷阱

### 秘密房间的入口（已被多次写错）
- **禁止**: "走廊尽头的门"、"那扇永远锁着的门"、"推门进入"、"门变成了XX"
- **正确**: "她们翻窗进入秘密房间"，房间没有门，只有窗户。世界重叠时房间**内部**会变形，但进入方式永远是翻窗
- 如果写"门"，故事就崩了

### 道具追踪
- `写作规范与一致性手册.md` 精确记录了善良公主口袋里有什么、白菜棋子备用数量等
- 每集结束后口袋里会多一个新物品（纪念品），必须写明
- 白菜棋子用量已在第7集中耗尽概念，须以手册为准

### 对话规则（为TTS语音朗读优化）
- **禁止**连续多人来回对话（A→B→A→B→A→B），听众分不清谁在说话
- **推荐**: 转述式、"A告诉B，XXX。B听完之后做了Y动作"、单人说话+叙述穿插
- 允许短对话，但不要超过两轮

### 打屁巫师的机器
- 每集一台新机器，名字有搞笑/屁元素，通常叫"XX仪"
- 关闭方式：需要至少两种来自不同世界的"核心物品"对抗
- 每集结尾狼狈逃跑，从不被真正抓住
- 云端图纸已被魅塔骑士删除（第4集）

## 文风要点
- 对标"不一样的卡梅拉"：大胆、天马行空、不低幼、不说教
- 允许适度粗俗（"屁""死""杀""尿裤子"等边缘试探）
- 孩子对话保持本真：直率、互相拆台、死缠烂打式追问
- 禁止小大人式对话、禁止过度解释角色背景

## 质量下降问题（已知）
上一个LLM因上下文窗口限制，后期故事出现以下问题：
- 秘密房间入口写错（把窗写成门）
- 道具数量前后矛盾
- 故事长度显著缩短、对话减少、叙述变机械
- **对策**: 每次续写时重新读取 `写作规范与一致性手册.md`，把关键设定放入当前上下文窗口，不要依赖记忆
