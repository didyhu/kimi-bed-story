# 《四象天灾阵》四集连续故事 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 撰写、审核并发布睡前故事《四象天灾阵》系列（第23-26集），解决森林环境控制仪大危机。

**Architecture:** 以B组（六小只）和打屁巫师为核心，采用“伙伴被困”链条式悬念推进，前三集每集关闭一台机器并导致一名角色被俘，最终集通过打屁巫师燃尽魔力的牺牲成功破局。

**Tech Stack:** Markdown 纯文本、deepseek-pro 审核子代理。

## 全局约束

*   **字数限制**：每集故事正文字数必须 $\ge 2000$ 字。
*   **口癖限制**：每集叙事中“不是X，是Y”口癖 $\le 4$ 次（目标 0-1 次），严禁“不是A也不是B而是C”。
*   **对话控制**：避免连续多人对话；任何描述旁白不得连续超过 2 段，必须穿插台词。
*   **机器关闭**：关闭机器必须使用至少 2 样本集开始前已有的口袋道具，且有牺牲/障碍。
*   **互动描写**：新纪念品放入口袋时，必须与至少 2 件已有物品产生互动描写。
*   **事后更新**：定稿后必须更新 `写作规范与一致性手册.md` 道具状态及 `故事历史记录.md`（事实、伏笔、出场率、口癖表）。
